    <div class="s-prose js-post-body" itemprop="text">
                    
<p>I am trying to read the lines of a text file into a list or array in 
python.  I just need to be able to individually access any item in the 
list or array after it is created.</p>

<p>The text file is formatted as follows:</p>

<pre class="lang-py s-code-block hljs python"><code><span class="hljs-number">0</span>,<span class="hljs-number">0</span>,<span class="hljs-number">200</span>,<span class="hljs-number">0</span>,<span class="hljs-number">53</span>,<span class="hljs-number">1</span>,<span class="hljs-number">0</span>,<span class="hljs-number">255</span>,...,<span class="hljs-number">0.</span>
</code></pre>

<p>Where the <code>...</code> is above, there actual text file has hundreds or thousands more items.</p>

<p>I'm using the following code to try to read the file into a list:</p>

<pre class="lang-py s-code-block hljs python"><code>text_file = <span class="hljs-built_in">open</span>(<span class="hljs-string">"filename.dat"</span>, <span class="hljs-string">"r"</span>)
lines = text_file.readlines()
<span class="hljs-built_in">print</span> lines
<span class="hljs-built_in">print</span> <span class="hljs-built_in">len</span>(lines)
text_file.close()
</code></pre>

<p>The output I get is:</p>

<pre class="lang-py s-code-block hljs python"><code>[<span class="hljs-string">'0,0,200,0,53,1,0,255,...,0.'</span>]
<span class="hljs-number">1</span>
</code></pre>

<p>Apparently it is reading the entire file into a list of just one 
item, rather than a list of individual items.  What am I doing wrong?</p>

<p>You will have to split your string into a list of values using <code>split()</code></p>
<p>So,</p>
<pre class="lang-py s-code-block hljs python"><code>lines = text_file.read().split(<span class="hljs-string">','</span>)
</code></pre>
<p>EDIT:
I didn't realise there would be so much traction to this. Here's a more idiomatic approach.</p>
<pre class="lang-py s-code-block hljs python"><code><span class="hljs-keyword">import</span> csv
<span class="hljs-keyword">with</span> <span class="hljs-built_in">open</span>(<span class="hljs-string">'filename.csv'</span>, <span class="hljs-string">'r'</span>) <span class="hljs-keyword">as</span> fd:
    reader = csv.reader(fd)
    <span class="hljs-keyword">for</span> row <span class="hljs-keyword">in</span> reader:
        <span class="hljs-comment"># do something</span>
</code></pre>

<div class="answercell post-layout--right">
    
    <div class="s-prose js-post-body" itemprop="text">
<p>So you want to create a list of lists... We need to start with an empty list</p>

<pre class="lang-py s-code-block hljs python"><code>list_of_lists = []
</code></pre>

<p>next, we read the file content, line by line</p>

<pre class="lang-py s-code-block hljs python"><code><span class="hljs-keyword">with</span> <span class="hljs-built_in">open</span>(<span class="hljs-string">'data'</span>) <span class="hljs-keyword">as</span> f:
    <span class="hljs-keyword">for</span> line <span class="hljs-keyword">in</span> f:
        inner_list = [elt.strip() <span class="hljs-keyword">for</span> elt <span class="hljs-keyword">in</span> line.split(<span class="hljs-string">','</span>)]
        <span class="hljs-comment"># in alternative, if you need to use the file content as numbers</span>
        <span class="hljs-comment"># inner_list = [int(elt.strip()) for elt in line.split(',')]</span>
        list_of_lists.append(inner_list)
</code></pre>

<p>A common use case is that of columnar data, but our units of storage are the
rows of the file, that we have read one by one, so you may want to <em>transpose</em>
your list of lists.  This can be done with the following idiom</p>

<pre class="lang-py s-code-block hljs python"><code>by_cols = <span class="hljs-built_in">zip</span>(*list_of_lists)
</code></pre>

<p>Another common use is to give a name to each column</p>

<pre class="lang-py s-code-block hljs python"><code>col_names = (<span class="hljs-string">'apples sold'</span>, <span class="hljs-string">'pears sold'</span>, <span class="hljs-string">'apples revenue'</span>, <span class="hljs-string">'pears revenue'</span>)
by_names = {}
<span class="hljs-keyword">for</span> i, col_name <span class="hljs-keyword">in</span> <span class="hljs-built_in">enumerate</span>(col_names):
    by_names[col_name] = by_cols[i]
</code></pre>

<p>so that you can operate on homogeneous data items</p>

<pre class="lang-py s-code-block hljs python"><code> mean_apple_prices = [money/fruits <span class="hljs-keyword">for</span> money, fruits <span class="hljs-keyword">in</span>
                     <span class="hljs-built_in">zip</span>(by_names[<span class="hljs-string">'apples revenue'</span>], by_names[<span class="hljs-string">'apples_sold'</span>])]
</code></pre>

<p>Most of what I've written can be speeded up using the <code>csv</code> module, from the standard library.  Another third party module is <code>pandas</code>, that lets you automate most aspects of a typical data analysis (but has a number of dependencies).</p>

<hr>

<p><em>Update</em> While in Python 2 <code>zip(*list_of_lists)</code> returns a different (transposed) list of lists, in Python 3 the situation has changed and <code>zip(*list_of_lists)</code> returns <em>a zip object</em> that is not subscriptable.</p>

<p>If you <em>need</em> indexed access you can use</p>

<pre class="lang-py s-code-block hljs python"><code>by_cols = <span class="hljs-built_in">list</span>(<span class="hljs-built_in">zip</span>(*list_of_lists))
</code></pre>

<p>that gives you a list of lists in both versions of Python.</p>

<p>On the other hand, if you <em>don't need</em> indexed access and what you want is just to build a dictionary indexed by column names, a zip object is just fine...</p>

<pre class="lang-py s-code-block hljs python"><code>file = <span class="hljs-built_in">open</span>(<span class="hljs-string">'some_data.csv'</span>)
names = get_names(<span class="hljs-built_in">next</span>(file))
columns = <span class="hljs-built_in">zip</span>(*((x.strip() <span class="hljs-keyword">for</span> x <span class="hljs-keyword">in</span> line.split(<span class="hljs-string">','</span>)) <span class="hljs-keyword">for</span> line <span class="hljs-keyword">in</span> file)))
d = {}
<span class="hljs-keyword">for</span> name, column <span class="hljs-keyword">in</span> <span class="hljs-built_in">zip</span>(names, columns): d[name] = column
</code></pre>

<div class="answercell post-layout--right">
    
    <div class="s-prose js-post-body" itemprop="text">
<p><strong>This question is asking how to read the comma-separated value contents from a file into an iterable list:</strong> </p>

<p><code>0,0,200,0,53,1,0,255,...,0.</code></p>

<p><strong>The easiest way to do this is with the <code>csv</code> module as follows:</strong></p>

<pre class="lang-py s-code-block hljs python"><code><span class="hljs-keyword">import</span> csv
<span class="hljs-keyword">with</span> <span class="hljs-built_in">open</span>(<span class="hljs-string">'filename.dat'</span>, newline=<span class="hljs-string">''</span>) <span class="hljs-keyword">as</span> csvfile:
    spamreader = csv.reader(csvfile, delimiter=<span class="hljs-string">','</span>)
</code></pre>

<p><strong>Now, you can easily iterate over <code>spamreader</code> like this:</strong></p>

<pre class="lang-py s-code-block hljs python"><code><span class="hljs-keyword">for</span> row <span class="hljs-keyword">in</span> spamreader:
    print(<span class="hljs-string">', '</span>.join(row))
</code></pre>

<p><strong>See <a href="https://docs.python.org/3.6/library/csv.html" rel="noreferrer">documentation</a> for more examples.</strong></p>
